Data models and query languages

Data models

  1. Relational: when we need transactions and simple (to some extent) query language. Data is normalized.
  2. Document: when we don't need a strict schema and can store most of necessary data in one document. Connected documents are rare.
  3. Graph: when we don't need a strict schema and we have many connected items.

Query languages

  1. Declarative: easier to read (because it's close to a natural language) and optimize (because storage details are hidden).
  2. Imperative: harder to read (because it's close to a programming language) and optimize but these languages are more expressive than declarative ones.
