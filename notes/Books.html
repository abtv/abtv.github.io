Books

These are the books which I highly recommend for software engineers.

On development

1. Code complete →

Approaches to use for software construction.

2. Patterns of Enterprise Application Architecture →

How to organize business logic in your application.

3. Refactoring →

Code smells and refactoring techniques. Requires practice during reading.

4. Unit Testing Principles, Practices, and Patterns →

How to write code which tests your code.

5. Microservices Patterns: With examples in Java →

When and how to develop microservices. Covers essentials of DDD, CQRS, Saga.

On teamwork

1. Mythical Man-Month →

On costs of people onboarding and communication.

2. Peopleware →

Why people are more important for successful projects than technology.

3. Goldratt's Theory of Constraints →

Identify a bottleneck and don't waste limited resources.

4. Elastic Leadership →

Three levels of team maturity and how to grow your team.
