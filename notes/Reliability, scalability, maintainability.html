Reliability, scalability, maintainability

Required properties of data intensive applications

  1. Reliablility.
  2. Scalability.
  3. Maintainability.

Reliability properties

  1. Tolerance to user input errors.
  2. Tolerance to individual server outages.
  3. Tolerance to human mistakes.

Scalability properties

  1. Scaling each 10x users will likely require architecture changes.
  2. Response time of 99.9 percentile is usually a good enough measure of system responsiveness.
  3. Distinguish latency (server time) and response (server + roundtrip time) time.

Maintainability properties

  1. Easy to understand code.
  2. Easy to make changes.
  3. Easy to run in production (operations).
