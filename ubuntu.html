Issue 1: How to connect to the server via ssh

//Connect to the server via ssh
ssh user_name@your_ip
//Upload file via ssh from local host to remote one
scp path_to_local_file_or_dirs user_name@remote_ip:path_to_destination_dir
scp some_file.txt root@10.10.10.10:/root.myfiles/
//Download the file via ssh from the remote host to local one
scp user_name@remote_ip:path_to_source_file path_to_local_dir
scp root@10.10.10.10:/root.myfiles/some_file.txt ~/

Issue 2: Users management

//Changing password
passwd
//Creating a New User
adduser andrew
//Adding root Privileges
visudo
Scroll down until you find a section that deals with user privileges. It will look similar to this:
# User privilege specification
root ALL=(ALL:ALL) ALL
Add another line below it that follows the format:
# User privilege specification
root ALL=(ALL:ALL) ALL
andrew ALL=(ALL:ALL) ALL
After this is done, press CTRL-X to exit. You will have to type "Y" to save the file and then press "ENTER" to confirm the file location.

Issue 4: How to install Mongodb

//Mongodb
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 7F0CEB10
echo 'deb http://downloads-distro.mongodb.org/repo/ubuntu-upstart dist 10gen' | sudo tee /etc/apt/sources.list.d/mongodb.list
sudo apt-get update
sudo apt-get install mongodb-org
Now you can type 'mongo' in command line
//and controls:
sudo service mongod start
sudo service mongod stop

Issue 5: How to install and configure Nginx

sudo apt-get install nginx
sudo mkdir -p /var/www/news/public_html
sudo chown -R andrew:andrew /var/www/news/public_html
sudo chmod 755 /var/www
sudo nano /var/www/news/public_html/index.html
sudo cp /etc/nginx/sites-available/default /etc/nginx/sites-available/news
//Let's configure our site
sudo nano /etc/nginx/sites-available/news
sudo ln -s /etc/nginx/sites-available/news /etc/nginx/sites-enabled/news
sudo rm /etc/nginx/sites-enabled/default
sudo service nginx restart
