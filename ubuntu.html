How to manipulate files

  • cp - copy files and directories
  • mv - move or rename files and directories
  • rm - remove files and directories
  • mkdir - create directories
//Copy a single file
cp file1 file2
//Copy multiple files to a different directory
cp file1 file2 file3 directory
//Copy all the HTML files from one directory to another
cp *.html destination
//Rename file
mv filename1 filename2
//Move files to a different directory
mv file1 file2 file3 directory
//Remove a file
rm file
//Remove a directory
rm -r directory
//Delete directories with content
rm -r dir1 dir2
//Generate all the parents for the new directory
mkdir -p /var/www/example.com/public_html

Users management

//Changing password
passwd
//Creating a New User
adduser andrew
//Adding root Privileges
visudo
Scroll down until you find a section that deals with user privileges. It will look similar to this:
# User privilege specification
root ALL=(ALL:ALL) ALL
Add another line below it that follows the format:
# User privilege specification
root ALL=(ALL:ALL) ALL
andrew ALL=(ALL:ALL) ALL
After this is done, press CTRL-X to exit. You will have to type "Y" to save the file and then press "ENTER" to confirm the file location.

How to install and configure Nginx

Nginx stores its configuration files within the "/etc/nginx" directory.
Directory /etc/nginx/sites-available/ contains configuration files for each site
Create /etc/nginx/sites-available and /etc/nginx/sites-enabled and then edit the http block inside /etc/nginx/nginx.conf and add this line
sudo apt-get install nginx
sudo mkdir -p /var/www/news/public_html
sudo chown -R andrew:andrew /var/www/news/public_html
sudo chmod 755 /var/www
sudo nano /var/www/news/public_html/index.html
sudo cp /etc/nginx/sites-available/default /etc/nginx/sites-available/news
//Let's configure our site
sudo nano /etc/nginx/sites-available/news
sudo ln -s /etc/nginx/sites-available/news /etc/nginx/sites-enabled/news
sudo rm /etc/nginx/sites-enabled/default
sudo service nginx restart
